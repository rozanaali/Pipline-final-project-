1. Introduction

For this project, I used the Ames Housing dataset, which contains 2,930 residential property sales in Ames, Iowa. The dataset includes more than 80 features describing different aspects of each house, such as lot size, structure, quality, neighbourhood, and the final sale price.

The main goal of this project was to clean the dataset and prepare it for further analysis and modeling. During the process, I focused on identifying data quality issues, fixing them, and building a reusable cleaning pipeline that ensures the dataset is consistent and ready for the next phases of the project.

2. Cleaning Summary

Before performing any analysis, the dataset required several cleaning steps to address common data quality issues.
_____________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________

Data types:
I inspected the data types of the columns and corrected any inconsistencies to ensure they were appropriate for analysis. Making sure each column had the correct data type helps prevent issues during later stages of the pipeline.

_____________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________
Missing values:
Some columns contained a very high percentage of missing values. These columns were dropped because they provided little useful information and imputing such a large amount of missing data could introduce noise. For important numerical columns with missing values, I filled them using the median, since it is less affected by extreme values and helps preserve the distribution of the data. For categorical columns, I used the mode, which represents the most frequent category in the column. This ensured that the dataset no longer contained missing values.
_____________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________

Duplicates:
I checked the dataset for duplicate rows and removed them if present. Duplicate records can bias the analysis and lead to misleading results.
_____________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________

Outliers:
Outliers were detected using exploratory techniques such as boxplots. Instead of removing these extreme values entirely, I capped them at an upper threshold. This approach reduces the impact of extreme values while still preserving the observations in the dataset.
_____________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________________

Finally, I combined all cleaning steps into a single clean_data() function, allowing the process to be easily reused if the dataset needs to be cleaned again. After completing the cleaning pipeline, I saved the processed dataset as cleaned_data.csv, which will be used in the next phases of the project.
